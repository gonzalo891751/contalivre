NOTAS PARA IMPLEMENTAR (React/TS)

// Componentes sugeridos:

  • StepRT6View: Contenedor principal con State de RT6.
  • KpiSummary: Row de las 4 cards (recibe `data`).
  • NatureAccordion: El bloque A/P/PN.
  • AccountRow: Fila de cuenta con lógica de expansión de orígenes.
  • OriginSubTable: La tabla de anticuación interna.

// Data Shape (Interface):

interface RT6Account {
  id: string;
  code: string;
  name: string;
  isMonetary: boolean;
  isForeignCurrency: boolean; // Si es true, va a No Monetarias con badge
  origins: {
    date: string; // YYYY-MM
    amount: number;
    indexOrigin: number;
    indexClosing: number;
    coefficient: number;
    homogeneousValue: number;
  }[];
  totalHistorical: number;
  totalAdjusted: number;
}

// Lógica "Analizar Mayor":

1. Fetch de cuentas con saldo no nulo al cierre.
2. Clasificar por Plan de Cuentas (Caja/Bancos/Créditos = Monet.; Bienes/PN = No Monet.).
3. Para No Monetarias: Buscar cada movimiento que compone el saldo final y agrupar por mes (Anticuación).
4. Matchear Mes contra tabla de Índices (IPIM/IPC) para calcular Coeficiente.
